# 书籍创作核心指南 (Writing Rules)

在撰写《爸爸不能时刻在身边：写给未来你的安全架构》的内容时，必须严格遵守以下原则：

## 1. 作者背景设定限制
*   **绝对不要**涉及“我作为安全工程师的工作经验”、“我在企业做网络安全防护的项目经历”或“帮企业做安全规划”等内容。
*   **真实背景定位**：作者是一位**考取了 CISSP 认证的父亲**。书中运用的仅仅是 CISSP 的**理论知识体系和思维模式**，而非具体职场实战经验。提及专业背景时，点到“考取认证时的学习心得”即可。

## 2. 语言风格设定
*   **通俗易懂**：语言必须极其平易近人、亲切自然，就像是一位父亲在饭桌上或散步时给孩子讲故事、讲道理。
*   **专业术语控制**：专业词汇（如：零信任、可用性、访问控制）要**克制使用、点到为止**。每次出现都必须立刻使用**最简单的生活案例**进行“翻译”。绝不堆砌 IT 术语，绝不制造阅读门槛。

## 3. 目标读者设定
*   **主要受众**：未成年的孩子，以及其他希望教导孩子安全的家长（尤其是女儿的父母）。
*   **情感基调**：兼具逻辑的严密性与父亲的温情。目的是**赋予孩子识别和规避危险的能力（Empowerment）**，绝不能通过传播焦虑、恐吓来让孩子对世界产生心理阴影。安全知识应当像防火墙一样在后台安静运转，而不是时刻拉响警报。

## 4. 内容详实与实用性要求（强烈约束）
*   **内容必须丰富饱满**：绝对不能只停留在干瘪的理论说教上。篇幅不需要强求多长，但必须干货拉满。
*   **提供极其具象的操作细节**：在讲解每一个安全防御点时，必须给出能在生活中直接落地、直接照做的巧妙小方法。不要只说“你要锁好门”，而是要给出诸如“门把手上顶水杯报警法”、“利用电梯控制面板的盲区防御”、“过两遍马路假装系鞋带试探尾随者”等极具实操价值的“物理外挂”。让整本书成为一本真正的“保命实操手册”。
